4.TCL(Transaction control language)
--------
def: The process of combining all related operations into a single unit and executing them according to the 
rule "either all operations should be succeed or none should be succeed" is called Transaction management.

->A transaction is considered a single unit of work and follows the principle

"either all or none"

example: funds transfer

->consider a money transfer process between two accounts.
1.debit amount from the sender's account.
2.credit amount from the receiver's account.

->Both operations should be executed as a single transaction.
->if the amount is debited from sender's account successfully. but crediting the receiver's account fails.
it may lead to data inconsistency problems.

There are two types of transactions such as
1.Local transaction
2.Global transaction

1.Local transaction
-------------------
In local transactions, all operations of a transaction are executed in the same database.
example: Funds transfer from one account to another account where both accounts belongs to
         the same bank. 

2.Global transaction
--------------------
In global transaction , operations of a transaction are executed on different databases.
example: Funds transfer from one account to another account where both accounts belongs to
         the different banks.

ACID Properties
---------------
Atomicity: all or nothing
Consistency: The database remains in a valid state
Isolation: Transactions do not interface with each other
Durability: Committed changes are permanent

LOCK
----

def: Locking is a mechanism used by the database to control concurrent access data.
->it ensures that multiple users access the same data simultanoueluy the database 
maintains consistency, integrity and prevent conflicts.

1.shared Lock
-------------
def: A shared lock allows to transaction to read data.
->Multiple transactions can hold a shared lock on the same row.
-> no transaction can modify the row until all shared locks are 
released.

characteristics:
read only
multiple users can read simultaneously
blocks updates and deletes.


2.Exclusive Lock
-----------------
def: An Exclusive lock allows a transaction to modify data.
 while an exclusive lock exists:
-> no other transaction can read or modify that row.
-> only the transaction holding the lock can update/delete the data.

characteristics:
used for INSERT, UPDATE, DELETE
only one exclusive lock at a time
blocks both read (locking reads) and writes


3.Intent Lock
--------------
def: An intent lock is a table level lock that indicates a transaction intends to place
row-level locks.
-> It helps MySQL quickly determine whether a table-level lock conflicts with existing row locks.
->user do not usually create intent locks manually .In no DB manages them automatically.

real-life examples
imagine a library:
shared lock-> many students can read the same book at the same time.
exclusive lock->One student borrows the book, so nobody else can read or borrow it until it is returned.
intent lock-> The librarian lock marks that someone is about to read or borrow books from a shelf,
helping coordinate access without checking every book individually.
